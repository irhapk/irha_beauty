# Implementation Plan: User Authentication

**Branch**: `003-auth` | **Date**: 2026-02-28 | **Spec**: [spec.md](./spec.md)
**Constitution**: v3.0.0

---

## Summary

Add email + password authentication to the Irha Beauty backend. Users can register, log in, log out, and silently refresh their session. All tokens are stored exclusively in httpOnly cookies (never in response bodies). Auth reuses the existing `users` table (adding a `hashed_password` column via migration) and the existing `users` repository — no new tables required. A reusable `get_current_user` FastAPI dependency is introduced for all future domains that need auth protection.

## Implementation Design

### `app/core/security.py` — Security Utilities

Responsibility: all cryptographic operations. No FastAPI dependencies, no DB access — pure functions.

```
Functions:
  hash_password(plain: str) -> str
    → passlib CryptContext.hash(plain)

  verify_password(plain: str, hashed: str) -> bool
    → CryptContext.verify(plain, hashed)

  create_access_token(user_id: int) -> str
    → PyJWT encode({sub: str(user_id), type: "access", exp: now+30min}, JWT_SECRET, HS256)

  create_refresh_token(user_id: int) -> str
    → PyJWT encode({sub: str(user_id), type: "refresh", exp: now+7days}, JWT_SECRET, HS256)

  decode_access_token(token: str) -> dict
    → PyJWT decode(token, JWT_SECRET, algorithms=["HS256"])
    → raises jwt.ExpiredSignatureError if expired
    → raises jwt.InvalidTokenError if malformed
    → raises ValueError if type != "access"

  decode_refresh_token(token: str) -> dict
    → same as decode_access_token but validates type == "refresh"
```

### `app/core/deps.py` — Dependency Update

Add `get_current_user` alongside existing `get_db`:

```
async def get_current_user(request: Request, db: AsyncSession = Depends(get_db)) -> User:
  token = request.cookies.get("access_token")
  if not token:
    raise AppException(detail="Not authenticated", code="NOT_AUTHENTICATED", status_code=401)
  try:
    payload = decode_access_token(token)
    user_id = int(payload["sub"])
  except jwt.ExpiredSignatureError:
    raise AppException(detail="Token expired", code="TOKEN_EXPIRED", status_code=401)
  except (jwt.InvalidTokenError, ValueError, KeyError):
    raise AppException(detail="Not authenticated", code="NOT_AUTHENTICATED", status_code=401)
  user = await users_repository.get_user(db, user_id)
  if user is None:
    raise AppException(detail="Not authenticated", code="NOT_AUTHENTICATED", status_code=401)
  return user
```

### `app/auth/service.py` — Business Logic

```
register(db, data: RegisterRequest, response: Response) -> UserRead
  1. get_user_by_email(db, data.email) → if exists raise 409 EMAIL_ALREADY_EXISTS
  2. hashed = hash_password(data.password)
  3. user = create_user_with_password(db, data.full_name, data.email, hashed)
  4. _set_auth_cookies(response, user.id)
  5. return UserRead.model_validate(user)

login(db, data: LoginRequest, response: Response) -> UserRead
  1. user = get_user_by_email(db, data.email) → if None raise 401 INVALID_CREDENTIALS
  2. if not verify_password(data.password, user.hashed_password) raise 401 INVALID_CREDENTIALS
  3. _set_auth_cookies(response, user.id)
  4. return UserRead.model_validate(user)

logout(response: Response) -> None
  1. response.delete_cookie("access_token")
  2. response.delete_cookie("refresh_token")

refresh(db, request: Request, response: Response) -> UserRead
  1. token = request.cookies.get("refresh_token") → if None raise 401 INVALID_REFRESH_TOKEN
  2. payload = decode_refresh_token(token) → on error raise 401 INVALID_REFRESH_TOKEN
  3. user = get_user(db, int(payload["sub"])) → if None raise 401 INVALID_REFRESH_TOKEN
  4. _set_access_cookie(response, user.id)  [only access token, not refresh]
  5. return UserRead.model_validate(user)

_set_auth_cookies(response, user_id) [private helper]
  → set_cookie("access_token", create_access_token(user_id), ...)
  → set_cookie("refresh_token", create_refresh_token(user_id), ...)

_set_access_cookie(response, user_id) [private helper]
  → set_cookie("access_token", create_access_token(user_id), ...)
```

### Cookie Helper Pattern

def _set_cookie(response: Response, key: str, value: str, max_age: int) -> None:

### `app/users/repository.py` — New Function

```python
async def create_user_with_password(
    db: AsyncSession, full_name: str, email: str, hashed_password: str
) -> User:
    user = User(
        full_name=full_name.strip(),
        email=email.lower().strip(),
        hashed_password=hashed_password,
    )
    db.add(user)
    await db.commit()
    await db.refresh(user)
    return user
```

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| bcrypt timing leak in `INVALID_CREDENTIALS` | Always call `verify_password` even when user not found (dummy hash) to prevent timing-based email enumeration |
| Cookie not sent cross-origin | CORS `allow_credentials=True` + frontend must use `credentials: "include"` on every fetch |
| JWT_SECRET accidentally committed | `.env` is gitignored; `.env.example` has only placeholder value |
| Existing tests break due to User model change | `hashed_password` is nullable — existing create_user() calls don't need to set it |
